## Repository purpose

Official repository of ISO/TC 211 XML (XSD) and JSON schemas, served at https://schemas.isotc211.org/. Schemas are organized by standard number, part, module, and version. Interactive schema browsers are generated by the companion site repo (`schemas.isotc211.org`) using LutaML tools.

## Validation commands

```sh
# Validate both package manifests (checks YAML structure, required keys, file existence, duplicate names)
ruby tools/validate_manifests.rb

# List all referenced files (used by CI for xmllint/json.tool checks)
ruby tools/validate_manifests.rb --list-files
```

CI also runs `xmllint --noout` on all `.xsd` files and `python3 -m json.tool` on all `.json` files listed in the manifests.

## Directory layout

```
{standard}/{-part}/{module}/{version}/
  {module}.xsd              # Main schema entry point (3-letter lowercase name)
  *.xsd, *.sch             # Additional schemas, Schematron rules
  index.adoc               # Documentation
  examples/*.xml           # Example instances

{standard}/resources/
  transforms/, codelists/, schematron/, bundles/, tools/

json/{standard}/{-part}/{module}/{version}/
  *.json                   # JSON Schema definitions
  examples/*.json          # Example instances
```

## Package manifests

- `lxr_packages.yml` — XSD (LXR) packages, each grouping one or more modules into a single browser
- `ljr_packages.yml` — JSON (LJR) packages

Required keys per package: `name`, `title`, `standard`, `files`. Optional: `status` ("current"/"draft"/"historical"), `description`, `part`.

File paths in manifests are relative to repo root.

## Build dependencies

`Gemfile` defines Ruby gems for LXR/LJR package generation: `lutaml-xsd`, `lutaml-jsonschema`, `lutaml-model`, `moxml`, `canon`.

LXR builds use `lutaml-xsd` with a shared template (`tools/lxr_build_template.yml`) that provides ISO/TC 211 branding, schema_location_mappings (to resolve URL imports to local files), namespace mappings, and appearance config. The CI generates per-package configs by merging the template with each package's file list.

LJR builds use `lutaml-jsonschema` to generate SPA documentation from JSON schemas.

## CI pipeline

`.github/workflows/ci.yml` runs on every push/PR:

1. **validate** — manifest validation, XSD well-formedness, JSON well-formedness
2. **setup-matrix** — parses `lxr_packages.yml` and `ljr_packages.yml` into JSON matrices
3. **build-lxr** (matrix) — builds `.lxr` packages using `lutaml-xsd build from-config` with `--validate`
4. **build-ljr** (matrix) — generates SPA docs using `lutaml-jsonschema spa`
5. **trigger-deploy** — on push to `main`, dispatches rebuild to the site repo

Each built package is uploaded as a GitHub Actions artifact. Build tools (`lutaml-xsd`, `lutaml-jsonschema`) are checked out from their repos and installed via `BUNDLE_GEMFILE=Gemfile.ci` to avoid conflicting with the repo's own Gemfile.
